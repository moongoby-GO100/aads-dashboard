P0 문서 버전 1.0 · 작성 기준 2026-05-13 KST · 뉴톡 AI 패션 인플루언서 사업기획서 부속 문서

1. 범위와 전제

1.1 In-Scope

  • 뉴톡 AI 패션 인플루언서 모델이 출연하는 패션 숏츠(15~60초) 자동 생성
  • 인스타 릴스 / 유튜브 숏츠 / 틱톡 동시 배포 포맷
  • 뉴톡 상품 기반 코디 착용컷 → 동영상 변환 파이프라인
  • AI 카피라이팅: 대본, 자막, 해시태그, CTA 자동 생성
  • 음악/효과음 자동 매칭 및 라이선스 관리
  • 뉴톡V1 상품↔숏츠 태그 연결 (판매 전환 추적)
  • ShortFlow v3.0 10-Layer 양산형 회피 엔진 적용

1.2 Out-of-Scope

  • 실사 촬영 영상 편집 (향후 Phase 2)
  • 라이브 커머스 연동
  • 유튜브 채널 자동 운영 (ShortFlow SaaS 영역)
  • 결제/과금 시스템 (별도 기획)

1.3 핵심 전제

  • AI 모델 이미지: AI 모델 생성/관리 기획서의 3-Layer 시스템에서 생성된 모델컷/코디컷 사용
  • 코디 매칭: 코디 매칭 기획서에서 확정된 코디 결과물이 입력
  • 상세페이지: 상세페이지 자동생성 기획서와 병행 — 동일 상품의 상세페이지+숏츠가 함께 생성
  • ShortFlow 기술 자산: SF 프로젝트의 10-Layer 엔진, FFmpeg 파이프라인, 2-Track 구조 활용
  • 뉴톡 DB 실측: shorts 10건, products 77,349건, sourcing_images 45건, categories 11개 [DB 조회, 2026-05-13]

유형구조예시 (모델: 하윤 22세)

대본 생성 규칙

  • 모델별 금지 표현 자동 필터링 (페르소나 카드의 forbidden_expressions)

  • CTA 패턴 5종 이상 로테이션 (동일 CTA 3회 연속 사용 금지)
  • 해시태그 자동 생성: 상품명 + 카테고리 + 트렌드 + 모델명 (최대 15개)
  • AI 생성 표시 문구 의무 삽입: "AI가 생성한 콘텐츠입니다"

출력: script (대사 텍스트, 자막 텍스트, 해시태그, CTA, 예상 타임라인)

모델컷 + 제품컷 + 감성컷을 대본 타임라인에 맞춰 시퀀스로 배열합니다.

  • 마감: CTA 텍스트 카드 + 상품 태그 오버레이

출력: image_sequence[] (이미지 경로, 표시 시간, 크롭 좌표, 전환 유형)

⑨ 품질 검증 — 10-Layer 엔진 적용

ShortFlow v3.0의 10-Layer 양산형 회피 엔진을 패션 숏츠에 맞게 커스터마이즈합니다.

Layer검증 항목패션 숏츠 기준실패 시 처리
L1대본 다양성동일 모델 최근 10편과 대본 유사도 < 0.70대본 재생성
L2시각 패턴컷 구성 순서·비율이 최근 5편과 ≥ 30% 상이시퀀스 재배열
L3오디오 변이동일 BGM 최근 3편 내 미사용BGM 교체
L4원본성 점수ORIGINALITY_SCORE ≥ 75자동 생성 중단, 수동 검토 대기
L5크로스 비디오 유사도전체 포트폴리오 대비 유사도 < 0.80모션/전환 효과 교체
L6정책 키워드플랫폼별 금지어 + 과장 광고 표현 필터해당 문구 자동 교체
L7발행 간격동일 모델 숏츠 발행 간격 ≥ 4시간발행 큐 지연
L8채널 다양성주간: OOTD ≤ 50%, 포커스 ≤ 30%, 기타 ≥ 20%유형 강제 교체
L9메타데이터제목·해시태그 중복도 < 0.60, 해시태그 5~15개자동 재생성
L10최종 품질해상도 1080×1920, 비트레이트 ≥ 4Mbps, 오디오 AAC 44.1kHz재렌더링

통과 기준: 10개 레이어 전체 PASS → 자동 발행 큐 진입. 1개라도 FAIL → 해당 레이어 자동 수정 → 재검증 (최대 3회). 3회 실패 → 수동 검토 대기.

  • 뉴톡 피드에서 숏츠 재생 시 상품 카드 오버레이 → 상품 상세페이지 링크
  • 전환 추적: 숏츠 조회 → 상품 클릭 → 구매 퍼널 측정

배포 스케줄링

  • 골든 타임: 평일 12:00~13:00, 18:00~21:00 / 주말 10:00~12:00 (KST)
  • 플랫폼별 최적 시간대 자동 계산 (추후 A/B 테스트 기반 학습)
  • 발행 큐: 생성 완료 → 품질 검증 통과 → 스케줄 큐 → 예약 시간 도달 → 자동 발행

출력: 발행된 URL, 상품 태그, 전환 추적 코드, 발행 시각

4.2 렌더링 흐름

  ↓

  ↓
무음 영상 + BGM + 효과음 → [amix 오디오 믹싱] → 음악 포함 영상
  ↓
음악 포함 영상 + 자막 ASS → [subtitles 필터] → 자막 포함 영상
  ↓
자막 포함 영상 + 워터마크 PNG → [overlay 필터] → 최종 MP4
  ↓
최종 MP4 → [qt-faststart] → 스트리밍 최적화 완료

7. ShortFlow 자산 활용 계획

SF 프로젝트(ShortFlow v3.0)는 이미 YouTube Shorts 자동화 SaaS로 설계되어 있습니다. 뉴톡 AI 패션 숏츠에서 직접 활용하거나 참조할 수 있는 자산을 정리합니다.

SF 자산활용 방안커스텀 필요 사항
10-Layer 양산형 회피 엔진 패션 숏츠 품질 검증에 직접 적용 패션 도메인 기준 추가 (L2 시각 패턴 → 코디 구성 검증, L6 → 과장 광고 필터)
2-Track 파이프라인 Track A(AI 생성) 위주로 활용, Track B(원본 편집)는 향후 실사 촬영 시 입력을 쿠팡 상품에서 뉴톡 상품으로 변경
FFmpeg 렌더링 파이프라인 동일 서버(114)에서 공유 모션 효과 12종 추가, 패션 전용 전환 효과
Worker + Redis 큐 숏츠 생성 작업 큐 인프라 재사용 뉴톡 전용 큐 채널 분리
TTS 엔진 연동 나레이션이 필요한 S3/S4 유형에 활용 모델별 음성 매핑 (페르소나 카드 기반)
YouTube API 연동 유튜브 숏츠 자동 업로드에 직접 활용 뉴톡 전용 채널 OAuth 설정 추가
