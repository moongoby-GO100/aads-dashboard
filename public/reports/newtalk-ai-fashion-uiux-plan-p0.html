요약

P0 UI/UX의 핵심은 "AI 모델을 보여주는 화면"이 아니라 상품을 선택하면 AI 모델 코디, 상세페이지, 숏츠, 판매 카드가 한 흐름으로 생성되고 승인되는 작업대를 만드는 것입니다. 고객 화면에서는 AI 모델이 상품을 스타일링해 보여주고, 관리자 화면에서는 사람이 승인해야 할 지점만 빠르게 판단하게 합니다.

P0 화면 목표

운영자는 신규상품 제품컷을 선택하고, AI 모델 6명 중 적합한 모델을 고른 뒤, 코디 컷/상세페이지/숏츠/판매 카드 초안을 한 화면에서 생성합니다. 결과물은 AI 표시 문구와 판매 검수 상태를 통과해야 공개됩니다.

UX 원칙

운영자 우선

관리자는 반복 생성보다 승인과 수정에 시간을 써야 합니다. 화면은 상품 선택, 모델 매칭, 결과 비교, 승인 순서로 고정합니다.

판매 연결 우선

모든 콘텐츠 카드에는 상품, 가격, 옵션, 상세페이지, 숏츠, 판매 상태가 붙어야 합니다. 예쁜 이미지 저장소로 끝나면 안 됩니다.

AI 표시 기본값

AI 모델, AI 착용컷, AI 생성 숏츠에는 기본 표시 문구를 노출합니다. 숨김 옵션은 만들지 않습니다.

10대 후반 가드

10대 후반 페르소나는 만 19세 성인으로만 운영하고, 교복/고등학생/미성년 연상 표현을 생성 옵션에서 제거합니다.

관리자 작업 흐름

01

모델 선택

페르소나 6명 중 상품군과 고객 연령에 맞는 모델을 선택합니다. 만 19세 모델은 성인 캐주얼 상품에만 노출하고, 교복/학교 상황 연출은 생성 옵션에서 제거합니다.

02

상품 선택

신규상품, 이미지 있음, 가격대, 카테고리, 마진율, 재고 가능성으로 필터링합니다. V1 상품 이미지 경로가 있는 상품을 우선합니다.

03

코디 생성

제품컷을 기준으로 모델 착용컷과 세트 코디를 생성합니다. 생성 실패, 제품 형태 왜곡, 과도한 신체 강조는 reject 사유로 저장합니다.

04

상세/숏츠 생성

상세페이지는 V1 상품등록 기준 필드를 유지하고, 숏츠는 video_url, thumbnail_url, 상품 태그를 채워 판매 시스템에 연결합니다.

05

판매 공개

승인된 상세페이지와 숏츠만 공개합니다. 상품 상세, 숏츠, 브랜드 페이지, 모델 프로필에서 같은 product_id를 추적합니다.

적용 순서

주차 작업 산출물 완료 기준
1주차 관리자 IA와 모델 스튜디오 AI Fashion 메뉴, 모델 6명 관리 화면 페르소나/금지 연출/AI 표시 저장 가능
2주차 상품 큐레이션 보드 신규상품 제품컷 필터, 코디 후보 담기 product_id 기준 작업 생성 가능
3주차 코디 생성 작업대 생성 결과 비교, 승인/반려, reject reason 승인된 이미지 URL이 작업에 저장됨
4주차 상세페이지 빌더 V1 기준 모바일 상세 템플릿 html_content 저장 및 미리보기 통과
5주차 숏츠 생성/태그 숏츠 업로드, 상품 태그, 썸네일 숏츠 상세에서 상품 태그 조회 가능
6주차 고객 룩북/상세 노출 AI 모델 프로필, 룩북, 상품 상세 연결 고객이 모델 룩북에서 구매 CTA까지 이동

완료 기준

운영 가능

  • 관리자가 모델, 상품, 생성물, 승인 상태를 한 흐름에서 처리
  • 상품 1개 기준 상세페이지와 숏츠 초안 생성 시간이 10분 이하로 감소
  • 반려 사유가 누적되어 재생성 기준으로 활용

판매 연결

  • 상세페이지, 숏츠, 모델 룩북이 동일 product_id를 공유
  • 숏츠 상품 태그에서 상품 상세로 이동
  • AI 착용컷과 실제 제품컷이 명확히 구분

검수 필수

  • AI 표시 문구 누락 시 공개 불가
  • 만 19세 모델은 성인 표시와 미성년 오인 방지 가드 필수
  • 상품 정보, 가격, 옵션, 배송 조건은 기존 판매 기준과 일치해야 함

다음 개발 단계는 이 문서를 화면 구현 지시서로 전환해, 관리자 라우트와 API payload를 고정하는 것입니다. 우선순위는 모델 스튜디오, 상품 큐레이션 보드, 코디 생성 작업대, 상세페이지 빌더, 숏츠 태그 연결 순서입니다.
