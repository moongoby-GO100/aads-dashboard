상세 기획서

뉴톡V1 상품등록 자산(products 76,203건, V1 GoodsCode 이미지 폴더)과 AI 모델 코디컷·감성 이미지컷·상품 데이터를 결합해 상세페이지 HTML을 자동 생성하고, 검수·버전관리·V1 동기화까지 운영 체계로 만든다.

버전: P0 v1 작성: 2026-05-13 13:30 KST 대상: 뉴톡 신규등록 상품(MVP) 연결: 사업기획서 v1 · 사용자흐름 v2 · 코디 매칭 P0 출처: NTV2 DB 실측 / V1 파일경로 확인
요약 범위·전제 입력 데이터 블록 시스템 템플릿 생성 파이프라인 AI 카피 V1 연동 운영 콘솔 버전관리 API/DB 리스크/가드 비용 90일 MVP KPI ← 사업기획서 v1

요약

상세페이지 자동생성은 “LLM이 글을 써주는 것”이 아니라, 상품 데이터 + AI 모델컷 + 감성컷을 블록으로 조립하고 카피만 LLM이 채우는 컴포지션 시스템이다. 뉴톡은 products 76,203건과 product_details(html_content longtext) 테이블을 이미 보유하고 있어 저장·버전·V1 동기화 골격은 추가 마이그레이션 없이 운영 가능하다.

핵심 결정 (CEO 확정)

  • 이미지 원천: V1 GoodsCode 폴더(/data/files/goods/goodscode/img/{GoodsCode}/)
  • 카테고리 매핑: V1 GoodsCode prefix 기준 (t/bl/pt/sk/ops/jk/nt/cd/set/nb/acc/bag/ns)
  • 상세페이지 연동: B안 — V1 HTML을 NTV2가 자동 생성·푸시
  • MVP 범위: C안 — 신규등록 상품 중 선택 생성
  • 모델컷 수: 사용자 설정 가능(주상품 ~20장 / 부상품 ~10장, 기본값)

핵심 결과물

  • 블록 12종 + 카테고리 템플릿 6종 + 시즌 톤 4종
  • LLM 카피 생성기(상품-페르소나-시즌 기반)
  • 운영 콘솔: 일괄 생성 큐, 블록 에디터, 미리보기, 검수 워크플로
  • 버전 관리: product_details.version 사용, A/B 롤백
  • V1 sync: v1_goods_idx 기준 양방향, 충돌 시 V2 우선

주의 — DB 실측 (2026-05-13)

  • products: 76,203건 (실측, 신규)
  • product_details: 0건 — 신규 생성 대상
  • product_images: 0건 — V1 파일경로 활용
  • product_categories: 0건 — V1 prefix 매핑 사용

3. 블록 시스템 (12종)

상세페이지는 12종의 블록을 순서·선택적으로 조립한다. 각 블록은 자체 데이터 스키마와 LLM 프롬프트를 가진다.

B01 · Hero Block

필수

메인 모델 풀샷 1장 + 상품명 + 한 줄 헤드라인 + 가격. 모바일 첫 화면 안에 들어가야 함.

데이터: hero_image_url, product.name, headline, retail_price, sale_price

LLM 프롬프트: "상품명+페르소나 톤+시즌"으로 12자 이내 헤드라인 1개 생성

B02 · Story Block

권장

모델 1~2컷 + 감성 카피 2~3문장. 브랜드/페르소나 톤으로 “이 옷을 입은 그 사람의 하루”.

LLM: 페르소나 말투 사전 + 시즌 + 상황 키워드

B03 · Detail Cut Block

필수

제품컷 4~12장(V1 폴더). 소재·디테일·봉제·라벨 등. 각 컷에 1줄 캡션 자동 생성.

B04 · Model Cut Block

필수

코디 매칭 산출 주상품 모델컷 ~20장(설정 가능). 정면/측면/디테일/풀샷/앉은컷 등 구성 룰 적용.

B05 · Mood Block

권장

감성 이미지컷(인물 부재 가능). 분위기 전달, 톤 일관성용. 1~3장.

B06 · Coordi Block

권장

같은 모델의 다른 뉴톡 상품과의 코디 추천 3~6세트. 각 세트는 클릭 시 해당 상품 PDP 이동.

입력: 코디 매칭 산출 coordi_sets[], 각 set의 products[]

B07 · Spec Block

필수

사이즈표, 소재 구성, 컬러, 제조국, 신축성, 비침, 핏감(평가 5단계 슬라이더).

가드: 사이즈 측정값은 LLM 추정 금지 — V1 입력값만 사용. 미입력 시 “정보 없음”.

B08 · Size Guide Block

권장

카테고리별 한국 여성 일반 사이즈 가이드(상의 S/M/L, 하의 26/28). 모델 착용 사이즈 표기.

B09 · Care Block

옵션

세탁/관리 가이드(아이콘 4~6개). 소재 기반 룰 자동 매핑(면→찬물 세탁 등).

B10 · FAQ Block

옵션

FAQ 3~5개 자동 생성. 카테고리별 템플릿(사이즈, 배송, 교환).

B11 · AI Disclosure Block

필수(법무)

“이 페이지의 일부 이미지는 AI로 생성된 가상 모델입니다.” + 페르소나 ID + 생성일자.

법무 근거: FTC AI disclosure, Meta/TikTok AI 표시 정책, 한국 표시·광고법.

B12 · CTA Block

필수

장바구니/구매 버튼, 옵션 셀렉터(색상/사이즈), 재고 상태, 배송 안내. V1 결제 페이지로 deeplink.

6-2. 금지어 필터

  • 의료/효과: 다이어트, 슬리밍, 체형 교정, 치료, 의학적 효과 — 단어 단위 차단
  • 과장: 최고/최저, 유일, 100%, 무조건, 보장
  • 비교 광고: 타 브랜드명, "더 좋은", "X보다"
  • 미성년 연출: 교복, 학생, 유니폼, 어린 — 19세 페르소나 운영 가드
  • 실존 인물 유사: 실명, 연예인명, 닮은꼴 — embedding 차단

6-3. 출력 검증

길이

각 필드 최대 길이 초과 시 truncate + 재생성

금지어

매치 1건이라도 있으면 재생성(최대 2회)

일관성

페르소나 말투 변화 감지(임베딩 유사도 ≥ 0.75)

중복

같은 카테고리 최근 100건과 헤드라인 cosine ≥ 0.9면 재생성

7. 뉴톡V1 연동

CEO 확정 — B안: V1 HTML을 NTV2가 자동 생성하고 V1에 푸시.

7-1. 매핑 키

  • NTV2.products.v1_goods_idx ↔ V1.goods.goodsIdx
  • 이미지 경로: V1의 {GoodsCode} 폴더를 NTV2에서도 그대로 URL 참조
  • 카테고리: V1 분류 우선, 미설정 시 prefix 추론

7-2. 동기화 흐름

  1. NTV2가 상세페이지 html_content 생성/승인 완료
  2. V1 admin API POST /admin/goods/{idx}/description 호출 (1차)
  3. 실패 시 DB 직접 UPDATE (2차, 운영승인 후만 허용)
  4. V1 페이지 200 OK + 본문 길이 ≥ 5KB 확인
  5. product_details.version 증가, sync_status=synced 기록
충돌 룰: V1에서 운영자가 수동 편집한 경우, NTV2 자동 푸시는 차단되고 운영자 알림. 운영자가 "V1 덮어쓰기" 옵션을 선택해야만 푸시 진행.

변경 유형버전 증분예

오타/소소 캡션 수정patch1.0.0 → 1.0.1
