1 개요 및 목표

스마트 커서 인스펙터는 CEO가 브라우저에서 AADS 대시보드(또는 임의 웹 앱)를 열고, 마우스를 UI 요소 위에 올리기만 하면 AI가 해당 요소의 디자인/접근성/일관성 문제를 자동 분석해서 화면 위 패널에 즉시 정리해 보여주는 기능입니다.

🎯 핵심 가치

별도 메뉴 이동 없이, 보고 있는 화면 위에서 바로 디자인 문제를 발견하고 수정 요청까지 이어짐

⚡ 즉시 피드백

커서 hover 300ms 후 DOM 정보 수집 → AI 분석 → 1~3초 내 결과 오버레이 표시

🔗 Design Studio 연동

발견된 문제에서 "수정 요청" 클릭 시 Design Studio 요청 카드가 자동 생성됨

🖥️ 화면 공유 기반

기존 ScreenShare 컴포넌트의 캡처 기능을 확장하여 커서 좌표 기반 영역 스크린샷 활용

목표 요약

구분목표
사용자 경험마우스 올리기만으로 디자인 피드백 → 클릭 한 번에 수정 요청 생성

6.2 ElementInspector 핵심 로직

// SmartCursor/ElementInspector.tsx — 핵심 pseudocode

function handleMouseMove(e: MouseEvent) {
  clearTimeout(debounceTimer);
  debounceTimer = setTimeout(() => inspectElement(e.clientX, e.clientY), 300);
}

async function inspectElement(x: number, y: number) {
  const el = document.elementFromPoint(x, y);
  if (!el || el === lastInspectedElement) return;

  const domInfo = {
    tagName: el.tagName,
    className: el.className,
    id: el.id,
    textContent: el.textContent?.slice(0, 200),
    rect: el.getBoundingClientRect(),
    parentChain: getParentChain(el, 3),
  };

  const styles = {
    computed: extractKeyStyles(getComputedStyle(el)),
    tailwindClasses: extractTailwindClasses(el),
  };

  const a11y = {
    role: el.getAttribute('role'),
    ariaLabel: el.getAttribute('aria-label'),
    tabIndex: el.tabIndex,
    contrastRatio: calculateContrastRatio(styles.computed),
  };

  const screenshot = await screenShareRef.current?.captureRegion(domInfo.rect);

  const result = await fetch('/api/v1/smart-cursor/analyze', {
    method: 'POST',
    body: JSON.stringify({ domInfo, styles, a11y, screenshot_b64: screenshot, cursor: {x, y} }),
  });

  setAnalysisResult(await result.json());
  setHighlightRect(domInfo.rect);
}

8 AI 분석 엔진 설계

8.1 2단계 분석 전략

비용 효율을 위해 규칙 기반 1차 → LLM 2차 계층 구조를 사용합니다.

단계방식속도비용대상
1차규칙 기반 검증기< 100ms$0색상 토큰 일관성, WCAG 대비율, 간격 패턴, 터치 영역, 버튼 패턴 일관성
2차LLM 시각 분석1~3초~$0.005/요청레이아웃 구조 판단, 정보 계층 분석, UX 카피 품질, 시각적 균형

8.2 규칙 기반 검증 항목

카테고리검증 규칙소스
색상하드코딩된 hex/rgb → 디자인 토큰 매핑 위반 탐지design_audit_service.py HEX_COLOR_RE, RGB_COLOR_RE
접근성WCAG AA 대비율(4.5:1), aria-label 누락, role 미지정프론트 수집 데이터 기반 계산
간격같은 컨테이너 내 자식 요소 간 margin/padding 불일치DOM parent_chain + computed style 비교
버튼같은 페이지 내 버튼 클래스 패턴 불일치design_audit_service.py BUTTON_CLASS_RE
터치인터랙티브 요소 최소 44x44px 미달rect 크기 검사
타이포font-size 계단(12/14/16/20/24)에서 벗어난 비표준 크기computed font-size 검사

8.3 LLM 분석 트리거 조건

  • → 사용자가 "상세 분석" 모드를 선택했을 때
  • → 1차 규칙 검증에서 문제가 0건일 때 (놓친 문제가 있을 수 있으므로)
  • → 복잡한 레이아웃 요소 (grid, flex 컨테이너)일 때
  • → 사용자가 "수정 요청 생성" 전에 상세 맥락이 필요할 때

8.4 LLM 프롬프트 구조

SYSTEM:
  당신은 UI/UX 디자인 전문 감사 AI입니다.
  주어진 UI 요소의 DOM 정보, 스타일, 접근성 상태, 스크린샷을 분석하여
  문제점과 개선안을 구조화된 JSON으로 반환합니다.

USER:
  ## 분석 대상
  - 요소: {tag_name} #{id} .{class_name}
  - 파일: {file_path}:{line}
  - 페이지: {page_url}

  ## 수집 정보
  DOM: {dom_info_json}
  스타일: {styles_json}
  접근성: {a11y_json}
  1차 검증 결과: {rule_based_results}

완료 판정 체크리스트 (Phase 1 MVP)

  • ☐ 채팅창에서 "🔍" 버튼으로 인스펙터 ON/OFF 토글 가능
  • ☐ 마우스 hover 시 대상 요소에 하이라이트 표시
  • ☐ 300ms debounce 후 분석 요청 발생
  • ☐ 우측 패널에 문제점/개선안/품질 점수 표시
  • ☐ 색상 토큰, 접근성 대비, 간격 일관성 중 2개 이상 규칙 작동
  • ☐ "수정 요청 생성" 버튼 동작 (Design Studio API 연동)
  • ☐ 인스펙터 OFF 시 오버레이 완전 제거, 기존 기능 영향 없음
  • ☐ 분석 응답 P95 < 3초
