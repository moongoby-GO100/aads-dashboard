1 개요 및 목표

스마트 커서 인스펙터는 CEO가 브라우저에서 AADS 대시보드(또는 임의 웹 앱)를 열고, 마우스를 UI 요소 위에 올리기만 하면 AI가 해당 요소의 디자인/접근성/일관성 문제를 자동 분석해서 화면 위 패널에 즉시 정리해 보여주는 기능입니다.

v2.0 확장: 풀스택 실시간 모니터링

디자인 분석을 넘어, 커서가 닿는 UI 컴포넌트의 백엔드 API 연동 상태, DB 연결/쿼리 성능, 코드 로직 건강도를 실시간으로 파악하고, 문제가 감지되면 선제적으로 보고합니다.

🎯 핵심 가치

별도 메뉴 이동 없이, 보고 있는 화면 위에서 바로 디자인 문제를 발견하고 수정 요청까지 이어짐

⚡ 즉시 피드백

커서 hover 300ms 후 DOM 정보 수집 → AI 분석 → 1~3초 내 결과 오버레이 표시

🔗 Design Studio 연동

발견된 문제에서 "수정 요청" 클릭 시 Design Studio 요청 카드가 자동 생성됨

🖥️ 화면 공유 기반

기존 ScreenShare 컴포넌트의 캡처 기능을 확장하여 커서 좌표 기반 영역 스크린샷 활용

📡 실시간 API 모니터링 v2.0

커서가 닿는 컴포넌트가 호출하는 백엔드 API의 응답시간, 에러율, 연결 상태를 실시간으로 표시

🗄️ DB 연결 상태 감시 v2.0

해당 API가 사용하는 DB 테이블의 연결 풀 상태, 쿼리 성능, 데이터 정합성을 확인

🔍 코드 로직 분석 v2.0

프론트 컴포넌트 → 백엔드 API → DB 테이블까지 전체 호출 체인을 역추적하여 로직 상태 표시

🚨 선제적 보고 v2.0

AI가 잠재 문제(느린 쿼리, 에러 급증, 메모리 누수 징후)를 감지하면 hover 전에도 해당 영역에 경고 표시

📱 시나리오 A: 간격 문제 발견

CEO가 채팅 목록 위에 커서를 올림 → AI가 "세션 항목 간 간격이 8px/16px 혼용, 12px로 통일 권장" 표시 → "수정 요청" 클릭 → 자동으로 요청 카드 생성

🎨 시나리오 B: 색상 비일관 탐지

버튼 위에 커서를 올림 → AI가 "이 버튼 #3b82f6, 같은 페이지 다른 버튼 #2563eb — 디자인 토큰 blue-500으로 통일 권장" 표시

♿ 시나리오 C: 접근성 경고

텍스트 영역 위에 커서를 올림 → AI가 "대비율 3.2:1 — WCAG AA 기준 4.5:1 미달" 경고 표시, 대비 개선된 색상 추천

📏 시나리오 D: 레이아웃 점검

카드 그리드 위에 커서를 올림 → AI가 "768px 이하에서 2열 → 1열 전환 누락, @media 쿼리 추가 권장" 표시

📡 시나리오 E: API 지연 감지 v2.0

세션 목록 위에 커서를 올림 → 시스템 패널에 "이 컴포넌트의 API /api/v1/chat/sessions 평균 응답 1,200ms (정상 범위 300ms 이하) ⚠️" → "→ 쿼리 최적화 요청" 버튼

🗄️ 시나리오 F: DB 연결 이상 v2.0

대시보드 메인 페이지를 열자마자 SSE로 "DB 연결 풀 사용률 85% — 자동 감지" 경고가 상단에 표시됨 → 클릭 시 상세 패널 열림

🔍 시나리오 G: 코드 로직 역추적 v2.0

에러가 표시되는 영역에 커서를 올림 → AI가 "이 컴포넌트 → GET /api/v1/ops/health → health_checker.py:_check_db() → pipeline_jobs 테이블" 호출 체인 표시

6.2 ElementInspector 핵심 로직

// SmartCursor/ElementInspector.tsx — 핵심 pseudocode

function handleMouseMove(e: MouseEvent) {
  clearTimeout(debounceTimer);
  debounceTimer = setTimeout(() => inspectElement(e.clientX, e.clientY), 300);
}

async function inspectElement(x: number, y: number) {
  const el = document.elementFromPoint(x, y);
  if (!el || el === lastInspectedElement) return;

  const domInfo = {
    tagName: el.tagName,
    className: el.className,
    id: el.id,
    textContent: el.textContent?.slice(0, 200),
    rect: el.getBoundingClientRect(),
    parentChain: getParentChain(el, 3),
  };

  const styles = {
    computed: extractKeyStyles(getComputedStyle(el)),
    tailwindClasses: extractTailwindClasses(el),
  };

  const a11y = {
    role: el.getAttribute('role'),
    ariaLabel: el.getAttribute('aria-label'),
    tabIndex: el.tabIndex,
    contrastRatio: calculateContrastRatio(styles.computed),
  };

  const screenshot = await screenShareRef.current?.captureRegion(domInfo.rect);

  // v1: 디자인 분석 + v2: 시스템 상태 병렬 요청
  const [designResult, systemResult] = await Promise.all([
    fetch('/api/v1/smart-cursor/analyze', {
      method: 'POST',
      body: JSON.stringify({ domInfo, styles, a11y, screenshot_b64: screenshot, cursor: {x, y} }),
    }).then(r => r.json()),
    fetch('/api/v1/smart-cursor/system-info', {
      method: 'POST',
      body: JSON.stringify({ component: identifyComponent(el), page_url: location.href }),
    }).then(r => r.json()),
  ]);

  setAnalysisResult(designResult);
  setSystemStatus(systemResult);  // v2.0
  setHighlightRect(domInfo.rect);
}

10 컴포넌트→API→DB 매핑 레지스트리 v2.0 NEW

스마트 커서가 UI 요소를 식별하면, 해당 요소가 어떤 백엔드 API를 호출하고, 그 API가 어떤 DB 테이블을 사용하는지 역추적해야 합니다. 이를 위해 컴포넌트→API→DB 매핑 레지스트리를 구축합니다.

10.1 레지스트리 구축 방식

방식설명정확도비용
1단계: 정적 분석프론트 코드에서 fetch()/axios 호출 추출 → 컴포넌트별 API 매핑 자동 생성80%1회성 스크립트

11.2 선제 보고 엔진 아키텍처

class ProactiveEngine:
    """30초 주기로 시스템 상태를 수집하고 이상 감지 시 SSE 푸시."""

    INTERVAL_SECONDS = 30
    THRESHOLDS = {
        "api_p95_ms": 1000,        # API 응답시간
        "error_rate_pct": 5.0,      # 에러율 %
        "db_pool_pct": 85,          # DB 풀 사용률 %
        "memory_pct": 85,           # 메모리 사용률 %
        "sse_reconnects_per_min": 3, # SSE 재연결 빈도
    }

    async def collect_metrics(self) -> dict:
        """health_checker + DB 메트릭 + API 메트릭 병렬 수집."""
        health = await health_checker.run_checks()
        db_stats = await self._query_db_stats()
        api_stats = await self._collect_api_metrics()
        return {"health": health, "db": db_stats, "api": api_stats}

    async def detect_anomalies(self, metrics: dict) -> list:
        """임계값 초과 항목을 선제 경고로 변환."""
        alerts = []
        if metrics["api"]["p95_ms"] > self.THRESHOLDS["api_p95_ms"]:
            alerts.append(self._make_alert("warning", "API 응답시간 급증", ...))
        if metrics["db"]["pool_pct"] > self.THRESHOLDS["db_pool_pct"]:
            alerts.append(self._make_alert("warning", "DB 연결 풀 포화", ...))
        # ... 각 임계값 체크
        return alerts

    async def push_to_sse(self, alerts: list):
        """감지된 경고를 SSE proactive/stream으로 push."""
        for alert in alerts:
            await sse_manager.send_event("system_alert", alert)

    async def run_loop(self):
        """메인 루프: 30초마다 수집 → 감지 → 푸시."""
        while True:
            metrics = await self.collect_metrics()
            alerts = await self.detect_anomalies(metrics)
            if alerts:
                await self.push_to_sse(alerts)
            await asyncio.sleep(self.INTERVAL_SECONDS)
