전체 사용자 흐름 개요

관리자(운영자)가 하나의 상품을 AI 패션 인플루언서 판매 콘텐츠로 완성하는 7단계 파이프라인입니다. 각 단계는 입력 → 처리 → 검수 → 다음 단계 진입 조건으로 구성됩니다. 어느 단계에서든 검수 탈락 시 이전 단계로 되돌아갈 수 있고, 최종 판매 반영은 운영자 승인 후에만 가능합니다.

핵심 원칙: 모든 AI 생성 결과는 초안(draft) 상태로 저장되며, 운영자 승인 후에만 승인(approved) → 판매반영(published)으로 전환됩니다.

상품등록

Product Registration

상품의 기본 정보를 등록하고 AI 제작 파이프라인을 활성화하는 첫 단계입니다. 기존 뉴톡V1 상품등록 필수값을 유지하면서, AI 모델컷/상세페이지 생성에 필요한 확장 필드를 추가합니다.

1-4. 서브스텝 상세

1-4-1. 상품 기본정보 입력

  • 상품명 입력 (중복 체크)
  • 상품코드 자동채번: {카테고리코드}-{연도}{월}-{시퀀스4자리}
  • V1 기존 상품이면 v1_goods_idx 자동 매핑
  • 공급가/판매가/도매가 입력, 마진율 자동 계산 표시

1-4-2. 옵션/재고 설정

  • 컬러별 사이즈 매트릭스 입력
  • 옵션별 추가가, 재고, 옵션코드
  • 0 재고 옵션은 AI 생성 대상에서 자동 제외
  • 컬러 헥스코드 입력 → 제품컷 색상 매칭 검증에 사용

1-4-3. AI 제작 옵션 활성화

  • "AI 모델컷 제작" 토글 ON
  • 타깃 연령, 시즌, 핏 키워드 선택
  • 금지 연출 입력 (선택)
  • AI 제작 큐에 등록 → ai_product_workflows INSERT

1-4-4. 진입 조건 체크

  • ✅ 상품명, 판매가, 카테고리 입력 완료
  • ✅ 옵션 1개 이상 등록
  • ✅ 재고 1개 이상 옵션 존재
  • ❌ 미충족 시: 필수값 누락 안내 → 저장은 되나 다음 단계 진입 차단
엣지케이스: V1에서 마이그레이션된 상품(v1_goods_idx 있음)은 기존 V1 이미지/상세페이지가 있을 수 있습니다. 이 경우 "기존 V1 데이터 가져오기" 버튼으로 제품컷 단계를 건너뛸 수 있으나, AI 생성 품질을 위해 제품컷 재등록을 권장합니다.

제품컷 등록

Product Cut Upload & QA

주상품의 실물 기준 이미지를 업로드하는 단계입니다. AI 모델컷의 품질은 여기서 올리는 제품컷의 품질에 직접 좌우됩니다. 최소 3장 이상 업로드해야 하며, 자동 QA를 통과한 이미지만 다음 단계 입력으로 사용됩니다.

업로드 즉시 자동 점검하여 100점 만점으로 채점합니다. 60점 미만은 자동 제외, 60~79점은 경고 표시, 80점 이상은 통과입니다.

2-3. 운영자 작업

대표컷 지정

드래그로 대표 이미지 1장 선택. product_images.is_primary = true 설정. 상세페이지 히어로, 검색 썸네일, 코디 보드에 사용.

옵션별 이미지 매핑

컬러 옵션이 여러 개면 각 옵션에 해당 컷을 연결. product_images.type에 "option_블랙", "option_아이보리" 등 기록.

불량컷 제외

QA 60점 이상이지만 운영자 판단으로 부적합한 컷은 수동 제외. 제외 사유 기록(색상왜곡, 구김, 오염 등).

보정 요청

해상도/색상 보정이 필요한 컷은 "보정 후 재업로드" 상태로 표시. 보정 전까지 AI 생성 입력에서 제외.

2-4. 데이터 흐름

입력: 이미지 파일 (JPG/PNG/WebP, 최대 10MB/장, 최대 20장)
저장: product_images 테이블 — product_id, type, path, filename, file_size, sort_order, is_primary
AI 확장: ai_product_assets 테이블 — workflow_id, asset_type='product_cut', qa_score, qa_details(json), status
출력: QA 통과 이미지 리스트 → ③모델선택, ⑤모델컷생성 단계의 입력으로 전달

2-5. 진입/탈출 조건

✅ 다음 단계 진입

QA 통과(60점 이상) 이미지 3장 이상 + 대표컷 1장 지정 완료

❌ 차단 조건

전체 이미지 QA 평균 60점 미만 / 대표컷 미지정 / 워터마크 이미지만 존재

모델선택

AI Model Selection (P0 수동 / P1 자동매칭)

등록된 상품에 가장 적합한 AI 모델을 선택하는 단계입니다. P0에서는 운영자가 6명 페르소나 카드를 비교해 직접 선택하고, P1부터는 상품 속성과 과거 성과 데이터를 기반으로 시스템이 최적 모델을 자동 추천합니다.

3-2. P1 자동매칭 점수 산정 공식

총점 = 카테고리 적합도(30) + 타깃 연령 매칭(20) + 스타일 키워드 일치(20) + 가격대 적합도(10) + 과거 성과(20)
항목배점산정 방법데이터 소스
카테고리 적합도30점 상품 카테고리가 모델 주력 카테고리에 포함되면 30점, 인접 카테고리면 20점, 무관하면 5점 categories + 모델별 주력 카테고리 매핑
타깃 연령 매칭20점 상품 target_age_range와 모델 페르소나 연령 차이 |Δ| ≤ 3세 → 20점, 4~7세 → 12점, 8세+ → 5점 ai_product_workflows.target_age_range + 페르소나 나이
스타일 키워드20점 상품 fit_keywords와 모델 style_keywords의 Jaccard 유사도 × 20 ai_product_workflows.fit_keywords + 페르소나 스타일 키워드
가격대 적합도10점 상품 판매가가 모델 담당 가격 범위 안이면 10점, ±20% 이내 7점, 범위 밖 3점 products.retail_price + 모델별 가격 범위
과거 성과20점 같은 카테고리+모델 조합의 과거 30일 CTR × 전환율 상위 → 20점, 중간 → 12점, 하위/데이터없음 → 5점 ai_generation_performance (P1에서 축적)

3-3. 복수 모델 선택

하나의 상품에 모델 2~3명까지 선택 가능합니다. 복수 선택 시 코디·모델컷·상세페이지가 모델별로 분기됩니다.

복수 선택이 유리한 경우

  • 연령층이 넓은 상품 (예: 기본 니트 → 서아+세린)
  • 스타일 범위가 넓은 상품 (예: 데님 자켓 → 루아+하린)
  • A/B 테스트로 어떤 모델이 전환율이 높은지 비교

단일 선택이 나은 경우

  • 명확한 타깃 상품 (예: 출근용 블레이저 → 서아)
  • 생성 비용/시간 제한이 있을 때
  • 첫 PoC 단계에서 변수 최소화

3-4. 가드레일

10대 후반 모델(박세린) 자동 차단 조건:
• 상품 카테고리가 "수영복", "속옷", "란제리"인 경우 자동 제외
• 금지 연출 키워드에 "교복", "미성년", "학생" 포함 시 자동 제외
• 운영자가 수동 선택해도 시스템이 확인 팝업으로 재확인

승인된 모델컷과 감성컷을 조합하여 뉴톡V1 상품등록 기준에 호환되는 상세페이지를 생성합니다. 단순 이미지 나열이 아니라, 판매 스토리 구조로 구성하여 조회 → 체류 → 장바구니 → 구매 전환을 최적화합니다.

7-2. 자동 초안 생성 로직

입력 데이터

  • 상품 기본정보 (products 테이블)
  • 옵션/재고 (product_options)
  • 승인 모델컷 (용도 태그별 분류)
  • 승인 감성컷 (타입별 분류)
  • 코디 세트 정보 (부상품 링크)
  • 모델 페르소나 (말투, 톤)

생성 프로세스

  • 1. 블록 순서에 따라 이미지 자동 배치
  • 2. 상품 속성 기반 판매 카피 자동 생성
  • 3. 모델 페르소나 말투로 추천 멘트 생성
  • 4. 사이즈 차트 자동 구성
  • 5. 배송/교환 템플릿 자동 삽입
  • 6. AI 표시 문구 자동 삽입

출력 형식

  • product_details.html_content에 저장
  • 모바일 퍼스트 반응형 HTML
  • 이미지는 CDN URL 참조
  • 블록별 JSON 구조도 병행 저장 (편집용)

V1 호환성

  • 기존 V1 상세페이지 렌더링과 동일한 HTML 구조
  • V1 이미지 정렬 도구와 호환
  • v1_goods_idx 있으면 V1 상품에도 연결 가능
  • 기존 상세 CSS 클래스 유지

7-4. 판매 반영 전 최종 체크리스트

✅ 필수 확인 (자동)

  • 판매가 입력 확인
  • 옵션/재고 1개 이상
  • 대표 이미지 1장 이상
  • 상세페이지 블록 5개 이상
  • AI 표시 블록 존재
  • 배송/교환 정보 존재

⚠️ 권장 확인 (수동)

  • 모바일 미리보기에서 이미지 잘림 없음
  • 판매 카피에 과장 표현 없음
  • 부상품 링크 클릭 시 정상 이동
  • 사이즈 차트 실측값 확인
  • 배송비 정확성

노출 채널 선택

  • ☐ 뉴톡 상품 상세
  • ☐ AI 모델 룩북 페이지
  • ☐ 브랜드 페이지
  • ☐ SNS 공유 링크 생성
  • ☐ 숏츠 연결

판매 상태 선택

  • 즉시 판매 (published)
  • 예약 판매 (날짜/시간 지정)
  • 비공개 (내부 검토용)
  • 품절 대기 (재고 입고 후 자동 오픈)

7-5. 판매 반영 후 성과 수집

판매 반영 24시간 후 자동 알림:
• 조회수, 상세페이지 체류시간, 스크롤 깊이
• 장바구니 추가율, 찜 수
• 구매 전환율, 객단가
• 모델별/코디 세트별 분리 추적
• 이탈 블록 (어디서 이탈했는지) → 상세페이지 개선 근거

데이터/API 설계

기존 뉴톡V2 테이블(products, product_images, product_options, product_details, categories)을 수정하지 않고, AI 제작 파이프라인 전용 테이블을 옆에 붙이는 방식입니다. 기존 판매 흐름을 깨뜨리지 않습니다.

P1 자동화 전환 조건

상품 10개 파일럿에서 아래 데이터를 확보한 뒤에만 자동매칭/자동추천 강화:
• 모델별 카테고리 승인률 통계
• 코디 세트별 클릭/전환 비교
• 프롬프트 버전별 폐기율 비교
• 상세페이지 블록별 이탈률 분석
데이터 없이 자동화부터 넣으면 잘못된 모델/코디가 대량 생성되어 검수 비용이 커집니다.
