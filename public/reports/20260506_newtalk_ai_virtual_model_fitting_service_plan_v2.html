1. 결론

뉴톡에 적용 가능하며, 단순 가상피팅이 아니라 AI 모델 스튜디오로 설계하는 것이 맞습니다. 제품 실사컷만 입력해도 모델 페르소나, 보유 상품 코디, 촬영 컨셉, 장소, 상세페이지 컷, SNS 숏츠까지 자동 기획·생성·검수되는 흐름입니다.

기술 선택은 패션 특화 FASHN Try-On/Product-to-Model/Model Create/Image-to-Video를 1차 엔진으로 쓰고, Nano Banana 2/Pro, Imagen 4, GPT-image-2, Veo 3.1을 보조 엔진으로 붙이는 하이브리드가 현실적입니다.

Phase 12주 MVP
운영자 생성·검수·저장
Phase 24~6주
대량 배치·상품상세 자동 반영
Phase 38~10주
판매자 셀프서비스·유료화

핵심 운영 원칙

  • AI 이미지는 자동 공개하지 않고 승인 후 공개합니다.
  • 상품의 소재·색상·로고·패턴이 바뀌면 불합격 처리합니다.
  • 모델 얼굴·체형은 뉴톡 소유 또는 라이선스 확보 모델만 사용합니다.
  • 대량 생성은 월정액 웹앱 수작업보다 API + 큐 + 캐시가 필요합니다.
  • 상세페이지에는 "AI 연출 이미지" 고지를 권장합니다.

2. v2 핵심 확장 — 7단계 AI 커머스 콘텐츠 흐름

1

AI 모델

다양한 페르소나의 전용 가상모델을 생성·보관합니다. 연령대, 체형, 분위기, 얼굴 노출, 시즌별 스타일을 모델 프로필로 관리합니다.

2

코디

뉴톡 보유 상품을 자동 매칭해 상의·하의·아우터·가방·신발·액세서리 조합을 만듭니다. 재고와 마진이 높은 상품을 우선 노출합니다.

3

촬영 컨셉

상품 카테고리와 타깃 고객에 따라 데일리룩, 오피스룩, 여행룩, 프리미엄룩, 시즌 캠페인 프리셋을 선택합니다.

4

장소

화이트 스튜디오, 거리, 카페, 쇼룸, 리조트, 실내 거울샷 등 장소 라이브러리를 적용합니다. 장소는 상품보다 튀지 않게 제한합니다.

5

판매 사진

썸네일, 대표컷, 착용컷, 디테일컷, 색상 비교컷을 생성하고 전환 점수 기준으로 후보를 정렬합니다.

6

상세페이지

승인 이미지를 기존 상세 이미지 정렬 플로우에 넣고, 카피·사이즈·소재·AI 고지 문구까지 묶어 상품 상세를 제작합니다.

7

SNS 숏츠

승인 컷을 3~8초 영상으로 변환하고 상품 매너, 릴스, 쇼츠, 스토리용 세로 영상 템플릿으로 배포합니다.

v2의 핵심은 "AI 이미지 생성 도구"가 아니라 상품 판매 목적의 콘텐츠 생산 공장입니다. 뉴톡 내부 운영자는 상품 판매력을 높이는 데 쓰고, 외부 판매자는 크레딧을 결제해 같은 파이프라인을 SaaS로 사용할 수 있습니다.

3. 서비스 컨셉

상품 판매자 관점

제품컷만 등록하면 모델 착용컷, 상세페이지 컷, SNS 컷을 빠르게 확보합니다.

가치 촬영비·모델섭외·스튜디오 시간을 줄입니다.

뉴톡 운영자 관점

상품 등록 후 AI 촬영 작업을 배치로 돌리고, 품질 점수와 후보 이미지를 보고 승인합니다.

가치 상품 상세 품질을 표준화합니다.

구매자 관점

제품 단독컷보다 착용 느낌, 길이감, 스타일링을 빨리 이해합니다.

주의 정확한 사이즈 피팅이 아니라 스타일 시각화입니다.

4. AI 모델·코디·컨셉 운영 설계

AI 모델 라이브러리

  • 기본 모델군: 20대 여성, 30대 여성, 미시/오피스, 캐주얼 남성, 스트릿 남성, 플러스 체형.
  • 페르소나 속성: 이름, 연령대, 키/체형, 얼굴 공개 여부, 피부톤, 헤어, 주력 카테고리, 금지 카테고리.
  • 일관성 관리: Face Reference 또는 기준 모델 이미지를 등록해 같은 모델이 여러 상품에 반복 등장하도록 관리.
  • 라이선스 관리: 모델별 사용권, 생성 허용 범위, 외부 SaaS 재판매 가능 여부를 DB에 기록.

코디 믹싱 엔진

  • 입력: 판매 상품, 보유 재고, 카테고리, 색상, 시즌, 가격대, 마진율, 판매 랭킹.
  • 출력: 단품 착용컷, 2개 상품 코디컷, 3~5개 상품 풀코디컷.
  • 제약: 품절/저재고 상품 제외, 브랜드 충돌 제외, 색상 조합 점수, 계절 부적합 제외.
  • 목표: 단일 상품 노출을 넘어 객단가를 올리는 세트 판매 이미지 생성.

촬영 컨셉·장소 엔진

  • 컨셉: 데일리, 출근, 데이트, 여행, 하객룩, 홈웨어, 골프, 프리미엄 쇼룸.
  • 장소: 스튜디오, 거리, 카페, 호텔, 리조트, 지하철 입구, 피팅룸 거울, 전신 거울.
  • 프롬프트 프리셋: 카메라 각도, 렌즈감, 포즈, 조명, 배경 밀도, 상품 강조 비율.
  • 금지: 배경 과장, 실존 브랜드 로고 노출, 제품보다 모델·장소가 튀는 구성.
운영 핵심은 "모델을 많이 만든다"가 아니라 상품군별로 잘 팔리는 모델·코디·장소 조합을 데이터화하는 것입니다. 생성 결과의 클릭률, 장바구니율, 구매전환율을 다시 프리셋 점수에 반영해야 시간이 갈수록 성능이 좋아집니다.

5. 판매 전환형 이미지 생성 기준

컷 종류 목적 생성 규칙 운영 지표
대표 썸네일 상품 목록 클릭 유도 제품이 화면 55~70%를 차지하고, 모델 얼굴보다 상품 실루엣이 먼저 보이게 구성. CTR, 스크롤 정지율, 찜 추가율
전신 착용컷 기장·핏 이해 정면/측면/45도 컷을 생성하고, 신체 비율 과장과 의류 변형을 제한. 상세 체류시간, 사이즈 문의 감소
코디컷 객단가 상승 뉴톡 보유 상품만 조합하고, 함께 구매 가능한 상품코드를 이미지 메타데이터에 연결. 세트 장바구니율, 연관상품 클릭률
디테일 보조컷 소재·포인트 강조 실제 제품컷을 우선 사용하고, AI는 배경 정리·톤 보정·확대 컷 생성에 제한. 반품 사유, 상품 문의, 리뷰 불만

6. 권장 사용자 흐름

1

제품컷 입력

기존 뉴톡 상품 이미지 또는 새 제품 단독컷을 선택합니다.

2

촬영 프리셋 선택

모델 성별·연령대·체형·배경·포즈·비율을 선택합니다.

3

AI 생성

상품당 6~12장 후보를 생성하고 실패 이미지는 자동 제외합니다.

4

QA 승인

색상·패턴·로고·핏·손가락·얼굴·배경을 체크합니다.

5

상세 반영

승인 이미지를 상세페이지 이미지 정렬/저장 플로우에 반영합니다.

뉴톡 V1의 기존 흐름인 /products/goods_img/{상품코드} 원본 이미지 등록과 /products/goods_img_sorting_test1/{상품코드} 상세 이미지 정렬 후 저장 구조에 "AI 착용컷 후보함"을 추가하는 방식이 가장 안전합니다.

파일럿 상품 100개 × 후보 6장 = 600장/월 FASHN Pro/Agency + 수동 QA 월정액/크레딧으로 검증 가능. 실제 단가는 계정 플랜에서 확인.
운영 시작 상품 1,000개 × 후보 6장 = 6,000장/월 FASHN API + Gemini Batch 보조 + 큐 처리 상품별 승인 이미지 2~3장만 보관해 저장/전송 비용 최소화.
대량 SaaS 상품 10,000개 × 후보 6장 = 60,000장/월 API 계약/볼륨 할인 + 자체 QA 자동화 + R2/CDN 월정액 웹앱으로는 운영 불가. API 볼륨 계약 필요.

고객: 상세페이지 노출

AI Commerce Content SaaS 외부 쇼핑몰/브랜드 API/웹 콘솔, 브랜드 모델 라이브러리, 대량 배치, CDN 저장, 숏츠 생성 월 구독 + 생성 크레딧 + 저장/전송량 + 팀 계정 수 기준.

SaaS 필수 기능

  • 테넌트별 모델 라이브러리, 상품 이미지, 생성 결과 완전 분리.
  • 브랜드별 금지 프롬프트, 로고/상표 정책, AI 고지 문구 커스터마이징.
  • 크레딧 지갑, 월 예산, 초과 사용 차단, provider별 원가 리포트.
  • 팀 권한: 소유자, 관리자, 디자이너, 검수자, 외부 대행사.
  • API 키, webhook, Shopify/Cafe24/자체몰 연동 어댑터.

뉴톡 내부와 SaaS의 차이

  • 뉴톡 내부는 상품 판매 성과가 KPI이고, SaaS는 생성량·유료 전환·재사용률이 KPI입니다.
  • 뉴톡 내부는 운영자 승인 중심, SaaS는 고객사별 브랜드 승인 워크플로우가 필요합니다.
  • 뉴톡 내부는 기존 상세페이지에 붙이고, SaaS는 다운로드/API/webhook을 모두 제공합니다.
  • SaaS는 테넌트 격리, 결제, 약관, 저작권·초상권 동의 로그가 P0입니다.
