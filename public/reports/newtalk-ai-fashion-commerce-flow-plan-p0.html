NewTalk AI Commerce Flow P0

뉴톡 AI 패션 인플루언서 실행 흐름 계획

AI 모델 생성에서 뉴톡 신규상품 코디, V1 기준 상세페이지와 숏츠 생성, 뉴톡 판매 시스템 판매까지 연결하는 P0 운영 설계입니다.

작성 기준: 2026-05-12 19:02 KST 대상: NTV2 / NewTalk 범위: AI 모델, 상품컷, 상세페이지, 숏츠, 판매 상태: P0 실행 계획
요약 4단계 흐름 1. AI 모델 2. 신규상품 코디 3. 상세/숏츠 4. 판매 데이터 연결 실행 일정 운영 가드

요약

P0 실행 흐름은 AI 모델 생성 → 뉴톡 신규상품 제품컷 기반 코디 → 뉴톡V1 상품등록 기준 상세페이지와 숏츠 생성 → 뉴톡 판매 시스템 판매 진행입니다. 핵심은 콘텐츠를 따로 만드는 것이 아니라, 상품 등록과 판매에 필요한 데이터 필드를 기준으로 AI 산출물을 생성해 바로 판매 전환에 붙이는 것입니다.

확정 방향

AI 모델은 한국 여성 10대 후반~30대 6명으로 운영하고, 10대 후반은 만 19세 성인 페르소나로만 제한합니다.

상품 기준

뉴톡 신규상품은 제품컷이 등록된 상품을 우선 사용합니다. 제품컷이 없으면 AI 착용컷 생성 대상에서 제외합니다.

등록 기준

V1 handoff 필드인 상품명, 도매처, 중국 URL, 원가, 옵션, 대표 이미지, 상세 이미지를 기본값으로 삼습니다.

개선 포인트

V1 기준은 등록 호환성을 보장하되, AI 코디컷, 사이즈/핏 설명, 모델별 스타일 문구를 상세페이지에 추가합니다.

4단계 실행 흐름

아래 순서가 P0의 기본 파이프라인입니다. 각 단계는 산출물이 다음 단계의 입력값이 되도록 설계합니다.

1

AI 모델 생성

확정된 6명 페르소나를 기준으로 얼굴, 체형, 헤어, 톤앤매너, 말투, 금지 콘셉트, disclosure 문구를 고정합니다.

  • 산출물: 모델 프로필, 기준 이미지, 금지선, 콘텐츠 말투, 상품 매칭 기준
  • 완료 기준: 같은 모델이 반복 생성돼도 얼굴과 분위기가 흔들리지 않는 reference set 승인
2

뉴톡 신규상품 제품컷으로 코디

제품컷이 등록된 신규상품을 모델별 스타일 축에 매칭하고, 단품컷을 코디컷, 착용컷, 룩북 컷으로 확장합니다.

  • 산출물: 모델별 상품 매칭표, 코디 세트, 대표컷, 보조컷, 제외 상품 사유
  • 완료 기준: 상품 원본 이미지와 AI 코디컷이 명확히 연결되고, 실제 판매 상품과 다른 구성품이 섞이지 않음
3

V1 기준 상세페이지와 숏츠 생성

뉴톡V1 상품등록 handoff 기준을 지키면서 상품 상세 HTML과 숏츠 대본/컷리스트를 생성합니다. 개선 가능한 영역은 AI 모델별 착용 설명, 핏, 코디 제안, 구매 CTA입니다.

  • 산출물: 상세페이지 HTML, 모델 착용 섹션, 상세 이미지 묶음, 숏츠 대본, 숏츠 상품 태그
  • 완료 기준: V1 등록 필드와 NTV2 Product/ProductDetail/ProductImage 구조에 들어갈 수 있는 형태로 정리
4

뉴톡 판매 시스템으로 판매 진행

상품 상세, 숏츠, 피드, 브랜드 페이지를 뉴톡 판매 흐름에 연결하고 모델별 성과를 상품 단위로 추적합니다.

  • 산출물: 판매 URL, 브랜드 페이지 노출, 숏츠 태그, 콘텐츠별 상품 링크, 전환 리포트
  • 완료 기준: 콘텐츠에서 상품 상세와 구매 흐름까지 이동 가능하고, 모델/상품/콘텐츠별 성과가 분리 집계됨

1. AI 모델 생성 계획

AI 모델은 판매를 위한 브랜드 자산입니다. 얼굴만 만드는 것이 아니라, 어떤 상품을 팔고 어떤 고객에게 말할지까지 한 번에 고정해야 합니다.

항목 생성 기준 검수 기준
기본 프로필 이름, 나이대, 직업/라이프스타일, 스타일 축, 타깃 고객, 가격대 실존 인물 유사 금지, 한국 여성 10대 후반~30대 범위 준수, 만 19세 성인 조건 준수
비주얼 reference 정면, 45도, 전신, 상반신, 표정 3종, 헤어/메이크업 기준 얼굴 일관성, 과도한 보정 금지, 미성년 오인 금지, 브랜드 톤 적합성
스타일 룰 선호 핏, 소재, 컬러, 계절, 착용 상황, 금지 상품군 모델마다 겹치지 않는 판매 역할 보유, 상품 매칭 근거가 설명 가능
말투와 콘텐츠 톤 피드 문장, 숏츠 내레이션, 상세페이지 추천 문구, CTA 과장 광고 금지, 허위 착용 경험 금지, AI 모델임을 숨기지 않음
표시 문구 "AI 패션 모델 이미지입니다", "뉴톡 판매 상품입니다" 등 고정 문구 게시물, 상세페이지, 숏츠 설명, 광고 소재에 누락 없이 삽입

10대 후반 모델 운영 제한

10대 후반은 만 19세 성인 페르소나로만 운영합니다. 교복, 고등학생, 미성년으로 보이는 외형/상황/말투, 보호자 동의가 필요한 듯한 설정은 금지합니다.

2. 뉴톡 신규상품 제품컷 기반 코디 계획

AI 코디는 제품컷이 등록된 뉴톡 신규상품에서 시작합니다. 상품 원본을 유지하면서 모델 착용 상황을 확장해야 판매 클레임을 줄일 수 있습니다.

단계 작업 입력값 산출물
상품 필터 신규상품 중 대표 제품컷, 가격, 옵션, 카테고리, 등록 상태가 있는 상품만 선별 products, product_images, options, categories AI 코디 가능 상품 리스트
모델 매칭 모델별 스타일 축과 상품 카테고리/가격대/컬러/시즌을 매칭 페르소나 카드, 상품명, 이미지, 가격, 옵션 persona_product_matches
코디 구성 단품을 상의/하의/아우터/신발/가방 조합으로 묶되 실제 판매 상품 외 구성품은 보조 소품으로 표시 대표컷, 상세컷, 모델 스타일 룰 코디 세트, 대표 착용컷, 보조 착용컷
품질 검수 상품 형태 왜곡, 색상 변경, 로고 생성, 실제 옵션과 다른 착용 이미지를 제거 원본 제품컷, AI 생성 이미지 승인 이미지와 반려 사유

코디 산출물 규격

대표 착용컷

상세페이지 상단과 피드 썸네일에 쓰는 1:1 또는 4:5 이미지. 상품 형태를 가장 명확히 보여줍니다.

상세 보조컷

핏, 소재감, 길이감, 코디 상황을 설명하는 이미지. 상세페이지 중단에 배치합니다.

숏츠 컷리스트

3초 후킹, 착용 전환, 디테일 컷, 가격/혜택, CTA 순서의 영상 구성표입니다.

3. 뉴톡V1 기준 상세페이지와 숏츠 생성 계획

NTV2의 ProductRegistrationService는 V1 상품등록 handoff를 위해 상품명, 중국 URL, 공급처, 가격, 옵션, 이미지 필드를 구성합니다. P0에서는 이 기준을 유지하면서 상세페이지 품질을 AI 커머스형으로 개선합니다.

V1 기준 필드/요소 현재 활용 방향 AI 개선 방향
GoodsName / title_kr 상품명과 한국어명 기본값 검색형 상품명 + 모델 추천형 서브카피 분리
GoodsEtc6 / supplier 도매처 기본값 "중국사입" 및 원 공급처 기록 상세페이지에는 공급처보다 소재/핏/활용 장면을 우선 노출
GoodsEtc7 / source_url 중국 원문 상품 URL 보존 관리자 검수용으로만 사용하고 고객 화면에는 노출하지 않음
GoodsPrice / 가격 계산 CNY, 환율, KRW 원가, 수수료, 마진 기반 판매가 계산 가격대별 CTA, 세트 코디 제안, 무료배송/혜택 문구 자동 분기
OptionColorChina / OptionSizeChina 색상과 사이즈 옵션 기본값 모델 착용 기준 사이즈 안내, 추천 핏, 옵션 선택 주의문 추가
main_image / detail_images 대표 제품컷과 상세 이미지 제품컷, AI 착용컷, 코디컷, 디테일 컷을 구분해 배치
ProductDetail.html_content 상품별 상세 HTML 저장 가능 구조 AI 모델 추천 섹션, 소재/핏 섹션, 숏츠 연결 섹션을 템플릿화

상세페이지 권장 템플릿

상단: 구매 판단

대표 제품컷, AI 모델 착용컷, 상품명, 핵심 가격, 옵션, AI 이미지 표시 문구를 한 화면에 배치합니다.

중단: 모델 코디 제안

선택된 AI 모델이 왜 이 상품을 추천하는지, 어떤 상황에 입는지, 함께 매칭할 상품군을 설명합니다.

하단: 상품 신뢰 정보

원본 제품컷, 상세컷, 소재/사이즈/배송/교환 안내, AI 생성 이미지와 실제 상품 차이 고지를 넣습니다.

숏츠 생성 기준

구간 내용 상품 연결
0-3초 모델별 후킹 문장과 착용컷 첫 화면 대표 상품 태그 1개
4-8초 착용 전환, 컬러/핏/길이감 강조 옵션 선택 주의점 연결
9-14초 코디 상황, 세트 상품, 가격대 장점 함께 살 상품 1-2개 태그
15초 이후 AI 모델 표시, 뉴톡 판매 상품 표시, 구매 CTA 상세페이지/브랜드 페이지 이동

4. 뉴톡 판매 시스템 판매 진행 계획

판매는 상품 등록 이후에 시작하는 별도 일이 아니라, AI 모델 콘텐츠 생성 시점부터 상품 태그와 전환 추적을 같이 붙이는 일입니다.

판매 접점 운영 방식 성과 지표
상품 상세페이지 AI 모델 착용컷과 원본 제품컷을 함께 노출하고 구매 버튼으로 연결 상세 진입, 옵션 클릭, 장바구니, 구매 전환
뉴톡 피드/브랜드 페이지 모델별 룩북, 코디 세트, 시즌 캠페인을 브랜드 페이지에 묶음 노출 저장, 공유, 브랜드 페이지 체류, 상품 클릭
숏츠 숏츠 상품 태그와 상세페이지 CTA를 연결해 발견형 유입을 만든다 완주율, 태그 클릭, 상세 진입, 구매 전환
외부 SNS Instagram/TikTok에는 AI 모델 및 광고/판매 표시를 포함하고 뉴톡 상세 URL로 유입 외부 유입, 랜딩 클릭, 신규 가입, 주문

P0 판매 시작 기준

첫 판매는 전체 6명 동시 론칭보다 모델 2명, 상품 50개, 숏츠 20개, 상세페이지 50개로 시작합니다. 14일 동안 저장률, 상세 진입률, 장바구니율을 보고 상위 모델과 상품군에 생산량을 집중합니다.

데이터 연결 설계

P0에서 개발해야 할 데이터 구조는 거창할 필요가 없습니다. 기존 상품, 이미지, 상세, 숏츠 태그 구조에 AI 모델과 생성 산출물의 관계만 붙이면 됩니다.

데이터 목적 P0 필드
ai_personas AI 모델 기본 정보 name, age_label, style_axis, disclosure_text, status
persona_product_matches 모델과 상품 매칭 persona_id, product_id, score, reason, reject_reason
ai_generated_assets AI 착용컷, 상세컷, 숏츠 컷 관리 persona_id, product_id, asset_type, image_url, prompt, approved_at
product_details 상품 상세 HTML 저장 product_id, html_content, version
short_product_tags 숏츠와 상품 연결 short_id, product_id
ai_disclosure_logs AI/광고/판매 표시 이력 content_id, channel, rendered_text, checked_at

P0 실행 일정

기간 작업 완료 산출물 판정 기준
1-3일 AI 모델 6명 reference set 생성 및 승인 모델별 기준 이미지 8장, 금지선, 말투, disclosure 문구 실존 인물 유사/미성년 오인/브랜드 부적합 0건
4-7일 신규상품 제품컷 기반 상품 50개 선별 모델별 매칭표, 제외 사유, 대표 제품컷 목록 제품컷 누락 상품 제외, 옵션/가격/카테고리 확인
